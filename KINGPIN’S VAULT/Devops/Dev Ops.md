# Software Development Life Cycle [SDLC]:

It is a process that is followed by the Industries to design , develop and test and finally give a high quality software

HLD'S:
- HLD focuses on "what" the system does
- It acts as a blueprint for the entire app, like defining user auth, database, and API layers for an e-commerce site.

LLD'S:
- LLD dives into "how" to build each part, created by developers with class diagrams, pseudo-code, and logic for modules.  
- It guides coding by specifying exact functions, error handling, and optimizations

## Roles in a company 
let example.com has branches like example_groceries , example_e-commerce etc...

in these lets take example_groceries [there are certain roles in this branch]:
- Business Analyst - They are directly connected to the customers and they get the rquirment from the cutomers. They will create the BRD's for each querry / requirement of the customers.

- Product Manager - They Interact with the BA and give prioitizes the BRD's ---> like this BRD should be completed within (JAN and FEB && this brd should be done first) [BRD - Business Requirment Document]

- Product Owner - Takes the prioritized BRD and breaks it into actionable items ---> UI required, Backend required, This particular DB required etc.. 
- ==actionable items - Epics==

- Software Architect - Creates the HLD'S AND LLD'S from the actionable items

The following roles will work togather : [scrum team]
- Developers - Development
- QE Engineers - Testing
- DBA - managing the database
- DevOps Engineers - Build CI / CD pipelines and manages the environment

- Release Managers (Optional)
- SRE
- Technical Writers - Will documentation of all works done
